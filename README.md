# 1. 우리 동네.zip

> 🏘 지역 주민들이 게시글, 모임, 건의사항을 통해 소통할 수 있는 지역 기반 커뮤니티 플랫폼

<br/>

## 📖 프로젝트 개요

우리 동네.zip은 지역 주민들이 동네 정보를 공유하고, 모임을 개설하며, 건의사항을 남길 수 있는 커뮤니티 서비스입니다.
게시판, 댓글, 좋아요, 모임 신청, 건의사항 등 커뮤니티 서비스의 기본 기능을 구현하며 웹 서비스의 전체적인 흐름을 경험했습니다.

<br/>

## 🗓 개발 기간

2026.03 ~ 2026.04

<br/>

## ✨ 주요 기능

* 회원가입 및 로그인/로그아웃
* 게시판 CRUD
* 댓글 및 좋아요 기능
* 카테고리별 게시판 이동
* 새 모임 만들기 신청 폼
* 동네 건의 페이지
* 공지사항 및 신고 기능

<br/>

## 🙋‍♀️ 담당 역할

* 메인화면 UI 구현
* 로그인/로그아웃 기능 구현
* 각 카테고리 페이지 네비게이터 구현
* 새 모임 만들기 신청 폼 구현
* 동네 건의 페이지 구현

<br/>

## 🛠 사용 기술

* Spring legacy
* JSP & Servlet
* HTML
* CSS
* JQuery
* Oracle DB

<br/>

## 🌱 배운 점

* MVC 구조를 기반으로 화면, 컨트롤러, 서비스, DAO의 흐름을 이해했습니다.
* 게시판, 로그인, 폼 처리 등 웹 서비스의 기본 기능 구현 경험을 쌓았습니다.
* 사용자 입력값 검증과 페이지 이동 흐름을 고려하며 기능을 구현했습니다.

# 2. SoSo

> 🧾 요식업 소상공인을 위한 재고·발주·거래처·정산 관리 통합 플랫폼

<br/>

## 📖 프로젝트 개요

SoSo는 요식업 소상공인이 재고, 발주, 거래처, 지출 내역을 한 곳에서 관리할 수 있도록 만든 경량 ERP 플랫폼입니다.
반복적인 발주 업무와 지출 관리를 효율화하고, RAG 기반 챗봇을 통해 매장 데이터를 쉽게 조회할 수 있도록 구현했습니다.

<br/>

## 🗓 개발 기간

2026.05 ~ 2026.06

<br/>

## ✨ 주요 기능

* 사업자 회원가입 및 로그인
* 매장별 재고 관리
* 거래처 관리
* 거래처 기반 발주 신청
* 발주 품목 기반 유사 재고 추천
* 카드 등록 및 발주 결제
* 지출 내역 등록, 수정, 삭제
* 월별 지출 금액 및 건수 통계
* RAG 기반 매장 데이터 조회 챗봇
* 공동 구매
* 재고/발주/공동 구매/영업일지 조회

<br/>

## 🙋‍♀️ 담당 역할

* 로그인, 로그아웃
* 아이디 찾기, 비밀번호 찾기
* 사업자 발주 관리(발주서 작성, 발주서 현황 확인)
* 사업자 매출/정산 관리(결제 관리, 비용 카테고리, 엑셀 내보내기)
* 거래처 수금 관리
* 챗봇

<br/>

### 🧾 발주 관리

* 거래처별 품목 조회 기능 구현
* 발주 신청 및 발주 내역 조회 기능 구현
* 발주 품목과 기존 재고의 유사 품목 추천 기능 구현
* 발주 완료 후 재고 입고 흐름 연동

<br/>

### 💳 결제 관리

* 카드 등록, 삭제 기능 구현
* 미결제 발주 목록 조회 기능 구현
* 선택한 발주 건에 대한 결제 처리 흐름 구현
* 결제 내역과 발주 데이터를 연결하여 관리

<br/>

### 💸 지출 관리

* 직접 구매 지출 등록, 수정, 삭제 기능 구현
* 일반 발주, 공동 발주, 직접 구매 지출 내역 분류
* 카테고리별 지출 등록, 수정, 삭제 기능 구현
* 월별 총 지출 금액 및 총 건수 집계
* 카테고리별 지출 통계 구현

<br/>

### 🤖 RAG 챗봇 연동

* 도메인 데이터를 문서화하여 RAG 검색에 활용
* 재고, 발주, 결제, 지출 데이터를 챗봇에서 조회할 수 있도록 구현

<br/>

* Spring Boot와 React를 연동하며 프론트엔드와 백엔드 간 API 통신 구조를 경험했습니다.
* RAG 챗봇을 구현하며 도메인 데이터를 기반으로 답변을 생성하는 흐름을 학습했습니다.
* GCP, Docker, Firebase Hosting 등을 활용하며 로컬 개발 환경과 배포 환경의 차이를 경험했습니다.
* 팀 프로젝트에서 문서화, 이슈 정리, 기능 분담을 통해 협업의 중요성을 배웠습니다.

## 💡 Project Experience Summary

두 번의 팀 프로젝트를 통해 웹 서비스의 기본 구조부터 실제 서비스 흐름까지 단계적으로 경험했습니다.

1차 프로젝트에서는 게시판, 로그인, 모임 신청, 건의사항 등 커뮤니티 서비스의 기본 기능을 구현하며 MVC 구조와 웹 개발의 기본 흐름을 익혔습니다.

2차 프로젝트에서는 재고, 발주, 결제, 지출, RAG 챗봇을 연결하여 하나의 서비스 흐름으로 완성하는 경험을 했습니다.
특히 사용자가 입력한 데이터가 데이터베이스에 저장되고, 이후 결제·지출·챗봇 응답까지 이어지는 흐름을 직접 구현하며 백엔드 데이터 처리의 중요성을 배웠습니다.
